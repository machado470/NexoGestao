# Guia de CI/CD para NexoGestão

Este documento detalha a configuração e o uso do pipeline de Integração Contínua (CI) e Entrega Contínua (CD) para o monorepo NexoGestão, utilizando GitHub Actions para automatizar testes, builds e deploys para ambientes de Staging e Produção.

## 1. Visão Geral do Workflow

O pipeline de CI/CD é composto por três workflows principais no GitHub Actions:

-   **CI (`ci.yml`)**: Executado em cada *push* e *pull request* nas branches `main` e `develop`. Realiza instalação de dependências, lint, typecheck, testes e build das aplicações `api` e `web`.
-   **Deploy Staging (`deploy-staging.yml`)**: Disparado automaticamente em cada *push* para a branch `develop`. Realiza o deploy da aplicação para o ambiente de Staging.
-   **Deploy Produção (`deploy-prod.yml`)**: Disparado **manualmente** a partir da interface do GitHub Actions. Realiza o deploy da aplicação para o ambiente de Produção.

## 2. Configuração de Secrets no GitHub

Para que os workflows de deploy funcionem, é necessário configurar as seguintes *secrets* no seu repositório GitHub. Acesse `Settings > Secrets and variables > Actions` no seu repositório.
